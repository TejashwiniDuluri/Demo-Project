Errors & Exceptions

As of now we have seen so many examples from them we encounter two different types of errors .They are : Syntax error and Exceptions.

Syntax Error

Syntax errors are the most common complaints you get while you are learning Python:

Example :

         >>for i in range(1,5)

                   print i

Output :

  File "<stdin>", line 1

    for i in range(1,5)

                       ^

SyntaxError: invalid syntax

The error is caused by the token preceding the arrow: in the example, the error is detected at the keyword print , since a colon  :  is missing before it.

Exceptions  Handling :

Errors detected during execution are called exceptions and are not unconditionally fatal. It implies that the problem doesn't occur frequently, i.e. the exception is the "exception to the rule".

Exception handling is a construct to handle or deal the error automatically. As Python and some other programming languages have bulit-in support for exception handling.

Bulit-in Exceptions :

Base class for all exceptions.

Exception

All user-defined exceptions should also be derived from this class.

StandardError

Itself is derived from exception.

Catching Exceptions

In python, try statement is used to handle exceptions. The operation which can be raise exception is placed in try clause and the code that handle the exception is written in except clause.

Example :

while True:

        try:

          n=input(“enter a number:”)

            n=int(n)

          break

       except:

              print “not a valid number”

print “entered successfully”

Output :

enter a number : 7

entered successfully

Example :

def catch(msg):

    try:

        raise ValueError(msg)

    except ValueError as e:

        if str(e)!="hello":

            raise

        else:

            print "caught!"

catch("hello")

Output :

caught!

- Here,in except ValueError, e, e is an instance of the exception, not a string. So when you test if e is not equal to a particular string, that test is always False. Try:if str(e)!="hello" instead.

Catching Specific Exceptions

A try clause can have any number of except clause to handle them differently but only one will be executed in case an exception occurs.

To specify multiple exceptions in an except clause,We can use a tuple of values.

Example :

try:

      age=input(‘enter your age:”)

except ValueError:

       print ‘invalid value’

except keyboardInterrupt:

        print ‘you interrupted the keys’

else:

          print ‘you are old’

        else:

          print ‘you are  young’

Output :

enter your age: 51

you are old

enter your age: 24

you are young

enter your age: a

invalid value

enter your age: cntrl+c

you interrupted the keys

Raising Exceptions

The raise statement allows the programmer to occur a specified exception.

The raise indicates the exception to be raised. This must be either an exception instance or an exception class

Syntax :

            raise [Exception [, args [, traceback]]]

Example :

try:

      a=input(“enter a number : ”)

      if a>=0:

        raise ValueError(“Negative number”)

except ValueError as ve:

      print ve

Output :

enter a number : -12

 Negative number

try…..finally

The finally clause is optional. It is intended to define clean-up actions that must be executed under all cases. A finally clause is always executed before leaving the try statement, whether an exception has occurred or not.

Syntax :

try:

      do operations here

      …………………..

  # if there is any exception this will be skipped

finally:

     This will be executed

Example :

try:

     x=input(        “enter a number :”)

     result=4*x

except KeyboardInterrupt:

     print “interruption”

else:

     print result

finally:

     print “finally”

Output :

enter a number : 5

20

finally

User-defined Exceptions

A user can create his own errors by creating a new exception class. Exceptions need to be derived from the exception class, either directly or indirectly.

Example :

class Sunny(Exception):

        def __init__(self, value):                                #__init__ Constructor or Initializer

                self.value = value

        def __str__(self):                                        # __str__ is to print() the value

                return(self.value)

try:

        raise(Sunny(10/5))

except Sunny as s:                                                # Value of Exception is stored in s

        print 'A new exception occurred : ', s.value

Output :

        A new exception occurred : 2

- we have created a user-defined exception called Sunny. which is derived from the Exception class. This new exception can be raised, using the raise statement and that value is stored in s ,when you get into exception then stored value is printed.

Errors Derived From Super Class Exception

Super class exceptions are used when module need to handle different types of errors. This can be done by creating a base class for exceptions defined by that module.

Example :

class MyException(Exception):

    pass

class ValidationError(MyException):

    def __init__(self, message, errors):

        self.message=message

        self.errors=errors

try:

    raise MyException({"message":"hiiii","fruits":["apple",”grapes”,”banana”]})

except MyException as e:

    details=e.args[0]

    print details["fruits"]

Output :

[‘apple’,’grapes’,’banana’]

Using standard exceptions as base class

Here,Runtime error is a class is a standard exception which is raised when a generated error does not fall into any category.

Example :

class ValidationError(RuntimeError):

    def __init__(self,arg):

        self.args=arg

try:

    raise ValidationError("hello")

except ValidationError as e:

    print e.args

Output :

('h', 'e', 'l', 'l', 'o')

-The above example illustrate, how to use runtime error as base class and Validation error as derived class.
