Object Oriented Concepts(OOP)

OOP refers to object oriented programming.Python is the mixture of both procedure oriented and object oriented concepts.The main difference between both is that in procedure oriented,the entire code is written in the form a procedure which is mixed form of data and logic,whereas in object oriented entire code is divided into self-contained objects and tiny programs. objects communicate among themselves for their functionality in the application.Thus oop concepts helps in creating reusable patterns of code,which in turn increases readability ,extensibility and maintainability.

Contents in object oriented programming includes the following:

Inheritance:

Inheritance plays a major role in object oriented programming which allows reusability of code.Inheritance allows a child class to inherit properties from the base class that is it includes both parent class properties and its own properties without effecting the parent class functionality.Thus it overcomes the problem of copying same code into child class from the parent class by simply inheriting from it.According to the terminology a class which is inheriting the properties from parent class is termed as derived class or child class or subclass and the class from which properties are being inherited is termed as base class or parent class or super class.

Parent class cannot access its child class members,but child class that is inherited from parent class can access the members of the parent class.Thus we can call both child class members and parent class members using child object.

Ex:a baby inherits all the properties of their parents other than her own properties.

*super keyword in inheritance:

super keword plays a major role in inheritance,when methods are ​overriden​(method with same name and same no.of parameters are declared in both parent and child classes)

any number of parent class methods can be called without explicitly using their names

MRO ​rule which says that child classes should be executed first than parent classes is followed in case of multiple inheritance.

Syntax:

super(subclass,instance).method(arguments)

example:

class father(​object​):

def meth(self,name): self.name=name return name

class daughter(father):

def meth(self,name1): self.name1=name1 return name1 super(daughter,self).meth(‘child1’)

class son(father):

def meth(self,name2): self.name2=name2 return name2 super(daughter,self).meth(‘child2’)

class gc(daughter,son):

def meth(self,name3): self.name3=name3 return name3 super(daughter,self).meth(‘grandchild’)

obj=gc()

obj.meth(‘micky’)

In the above example,all the child classes are executed first before parent class as per mro rule.Thus object to the gc class first called its own method and then super is encountered that called its first parent ​daughter and assigned ‘grandchild’ value to the instance variable of the daughter method again super is encountered but the flow goes to the next parent of ​gc rather than top parent because son child is still remained to execute.Hence son method is then executed and finally top parent class ​father is executed .

Flow of execution:

class Father(object):

def meth(self,name): self.name=name print name

class Daughter(Father):

def meth(self,name1): self.name1=name1 super(Daughter,self).meth("child1")

In the example above first ‘micky’ value is assigned to Gc class method instance variable that is name3 and is printed then after super is encountered,which assigns the value ‘grand_child’ to its first parent Daughter method instance varible that is name1 but is not printed ,again after encountering super ‘child1’ value is assigned to next parent of Gc class that is Son class method instance variable name2 and it is also not printed but again when super is encountered,the value ‘child2’ is assigned to the top parent class ​Father​method instance variable and is printed.

ABSTRACTION

Abstraction refers to hiding unnecessary data and functionality from user and providing him with only the required part thus simplifying complexity.for eg:if user wants to roast a bread in toaster it is enough to know about the ingredients that are needed to roast a bread rather than the internal functionality of toaster,thus the complexity of knowing unnecessary internal functionality of toaster is ignored.

In programming there are number of methods that simply perform different functionalities which are used as in_built methods without knowing their internal functionality.

For eg:

1)a ​print statement simply displays the things that are given to it by a user,where user is unaware of how it is displaying actually(internal functionality).

2)sum(2,3) ​method which adds up the numbers 2,3 given as arguments ,displays result by adding the both numbers as 5.Here user is able to see the method and its final result but not how it is actually performing it thus hididng the complexity and showing only necessary functionality reflects the property of abstraction.

There are basically two types of data members those are public and private.

Any variable or method except init() that needs to be declared as private is defined with the help of double underscore infront of it as __variablename and __methodname ,whereas public members are normally declared .

Private variables(hidden) are accessed only inside the class ,they can be still accessed outside the class using class methods which are existing inside the class or using class name.thus private variables are indirectly accessed.

Private methods are also represented with __ double underscore infront of it and access is also same like private varibles.private methods cannot be accessed directly using class object,they can be indirectly accessed using other class methods.

method overriding:

Method overriding refers to defining a method in parent and child classes with same name and same parameters but with their own different implementations.Hence such a method when called from child class it overrides the functionality of parent class method but when parent calls that method it returns its own implementation.both the methods are differentiated with the help of objects.i.e parent object is used to call parent method and child class object is used to call child class object.

For eg:

class mummy:

def punish(self,a):

#parent method

self.a=a

return a*a

class son(mummy):

def punish(self,a):

#child overriden parent action() method

self.a=a

return a+a

 

big=mummy()

#parent object

small=son()

#child object

big.punish(2)

#calling parent method

small.punish(3)

#calling overriden method will replace the parent
