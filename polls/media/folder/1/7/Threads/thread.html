MULTI-THREADING

Before going deeper into the concept of multi-threading,one has to know “what actually a process is?”.In computers point a program is one which consists of set of instructions in a sequence that perform certain task.Programs are executable files that reside inside harddisk(secondary memory) and are loaded into primary memory for execution based on the need hence this procedure of execution is stated as a process .simply a program under execution is referred to as process .Such a process is divided into certain no of threads each handling its own functionality that is assigned to it.

Thread:

  • A thread is referred to as a light weight process that  communicate with other threads residing in the same process and shares programmes resources like memory , code or instructions  and variable values like global variables.
  • Every process atleast contains one thread.
  • A thread contains states like new,ready,running,waiting,blocking,exit.
  • Every thread is associated with a pointer that keeps track of its move.
  • Every threads execution state has to be recorded while dealing with multiple threads paralelly.
  • Thread can be put into waiting state while other thread is running(sleep)
  • thread can be interrupted by other threads
  • a thread can consists of its own local variables other than global variables that are shared by all the threads inside a process.

There are basically two types of threads :

User threads:

        User level threads are those which are created and managed by user level library.these are anyhow executed by kernel but kernel is unaware of these threads management .These threads simply require a function call to create ,switch among themselves and synchronize.hence such type of threads are easy to handle and faster  than kernel level threads.here problem arises when any thread gets blocked because in user-level threads if one is blocked all the other threads are also blocked.Hence user level threads are suggested in non-blocking processes.

Kernel-level threads:

        kernel-level threads are those which are completely under kernels management and hence are operating systems dependent.The main advantage with these threads is that if one thread is blocked ,it doesnt not effect  other threads thus process runs as usual hence these type of threads are suggested in blocking type of processes.kernel-level threads are slower than user-level threads as they operates on different processors or cores.

Multi-threading

        process of executing multiple threads parelelly is referred to as multi-threading.multi-threading is better implemented on multi-processor systems  it can also be implemented on single-processor systems using thread scheduling .

  • The advantage is that every time cpu is kept under utilization and there is no hope for keeping cpu under idle state thus increasing performance.
  • Disadvantage with this approach is that in case of single processor systems managing multiple threads is an overhead.
  • When multiple threads tries to access to a single resource for longer time threads get into starvation,and there can also be a chance of  blocking threads and getting into deadlock state.

Implementing threads

        in python programming there are two modules that are used to implement threading.

  • <thread>: this module treats thread as a function
  • <threading>: this module is used in object oriented approach.every thread that is created is treated as an object.

        

<thread> module:

        syntax:

                thread.start_new_thread(function, args [,  kwargs])

         here thread implementation is function body and it terminates when function returns. Args and kwargs are arguments passed to the function in tuple form and are optinal.kwargs are the key value type of arguments and represents dictionary structure.

        Example:

                import time

                def pow(*args):

                            for i in args:

                                        time.sleep(1)

                                        print i**i

                def square(*args):

                            for i in args:

                                        time.sleep(1)

                                        print i*i

                

                t= time.time()

                pow(1,2,3)

                square(1,2,3)

                print time.time()-t

        o/p:

                1

                4

                27

                1

                4

                9

                6.00738883018

                

                

in the above example two  functions are declared such that one operates after other.when time.sleep(1) is encountered in first function,cpu waits for 1 sec and then proceeds to next iteration but not into next function .In this waiting time cpu is kept idle doing nothing .time.time()-t in above code prints the time that has been taken by two functions to execute the instructions,thus as shown around 6.00738883018 sec has been taken by the above program.

Only when first function has executed completly ,second function is executed thus resulting in decreased performance.now to increase the cpu utilization threading is been implemented as shown.

<threading>module:

        In this type of thread creation, every thread is considered as on object.

        Example:

                import time

                import threading

                from threading import Thread

                def powe(*args):

                                        time.sleep(1)

                                        print i**i

                def square(*args):

                           for i in args:

                                        time.sleep(1)

                                        print i*i

                t= time.time()

                t1=threading.Thread(target=powe,args=(1,2,3))

                t2=threading.Thread(target=square,args=(1,2,3))

                t1.start()

                t2.start()

                t1.join()

                t2.join()

                print time.time()-t

                1

                1

                4

                 4

                9

                 27

        as shown above before creating any thread <threading>nodule has to be imported from where Thread module is included.thread creation takes two parameters one is target that is function name and other is args which is list of function arguments.

Every thread inorder to implement its functionality it has to be started first using start() method which in turn triggers run() method and join() method waits until that thread execution is been completed.

        In class based approach inorder to implement threading following steps are followed up:

1) create subclass by extending the  main class Thread

2)__init__(self,[,args]) method of parent has to be overriden inorder to add up any additional parameters.

3)override run(self [,args]) method for thread implementation.

o/p:

  • In the above example ,class ‘me’ is derived from the main class Thread.There created 2 threads t,t1 that   represents the object creation .
  • start() method above calls the run() method everytime by default.it is called on every object once,if called more than once on same object it raises error.
  • here threading.currentThread() displays the currently running thread and getName() displays the name of the current thread.
  • run() method  above is overriding the run() method of Thread class which also accepts parameters inside it(args,kwargs)

Synchronization:

        synchronization is a mechanism where two threads or processes do not concurrently execute a section of code which is termed as critical section.everytime only one thread is allowed to enter into critical section and other threads which are waiting for the same resource has to wait until already entered thread comes out of it.Synchronization mechanism is achieved through the help of locks.

        When one thread wants to enter into that section of code it acquires lock such that other threads wont be allowed at the same time to execute that section of code.Other thread  execute it,only when first thread releases that lock.if first thread wont release that lock waiting thread goes into deadlock state .

lock=threading.Lock()  :  here lock is the object using which lock can be acquired and released

lock.acquire()  : lock is applied

lock.release()  :  lock is released

Methods:

        various methods provided in multi-threading are as follows:

start()  :  starts the thread execution and in turn invokes the run() method

run()  :  run is the entry method for any thread.

join([time])  :  It waits until thread gets completed its execution. 

getName()  :  it returns the name of a thread

setName()  :  it is used to set name to thread.If already set it updates the value.

isAlive()  :  ckecks whether thread is active or not.

threading.activeCount()  :  it returns the number of active(alive) threads.

threading.currentThread()  :  it returns the currently running thread.

threading.enumerate()  :  returns the number of currently active threads.
